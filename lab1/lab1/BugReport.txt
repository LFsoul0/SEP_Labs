# StudentWrapper类中，成员变量id和student均为const常量，student为引用。
# const常量只能被初始化，而不能被赋值。
# 引用也只能在初始化时指定引用对象，而对其赋值将被解释为对其引用对象赋值。
# 从概念上讲，调用类的构造函数时，对象（当然，包含其成员变量）将在函数体（大括号内）代码执行前被创建。
# 因此，原有构造函数将导致程序首先为id和student分配内存（不初始化），然后执行函数体代码，以常规赋值方法为id和student赋值，从而与const及引用的限定冲突。
# 改为使用成员初始化列表（member initializer list），可使id和student在执行构造函数体之前，即创建对象时进行初始化，从而避免上述问题。
